# 📘 Glossaire Technique - FortressLab

Ce document recense les termes techniques, commandes et concepts clés rencontrés durant l'autoformation en cybersécurité.

## A
- **ACL (Access Control List)** : Liste de contrôle d'accès définissant qui peut accéder à quoi.
- **Attacker Mindset** : Façon de penser visant à identifier comment abuser d'une fonctionnalité légitime.

## B
- **Bit SUID (Set User ID)** : Permission spéciale sous Linux permettant à un fichier de s'exécuter avec les droits de son propriétaire (souvent root) plutôt que de l'utilisateur qui le lance.
- **Brute Force** : Attaque consistant à essayer toutes les combinaisons possibles de mots de passe.

## C
- **CLI (Command Line Interface)** : Interface en ligne de commande (le terminal), opposée à l'interface graphique (GUI).
- **Chmod** : Commande Linux pour changer les permissions d'un fichier (`chmod 755 file`).
- **Chown** : Commande Linux pour changer le propriétaire d'un fichier (`chown user:group file`).
- **CIA Triad** : Les trois piliers de la sécurité : Confidentiality (Confidentialité), Integrity (Intégrité), Availability (Disponibilité).

## E
- **Exploit** : Code ou technique utilisée pour tirer parti d'une vulnérabilité.
- **/etc/passwd** : Fichier système listant tous les utilisateurs (lisible par tous).
- **/etc/shadow** : Fichier système contenant les hashs des mots de passe (lisible uniquement par root).

## G
- **Git** : Système de gestion de versions distribué.
- **GitHub** : Plateforme d'hébergement de code et de collaboration basée sur Git.
- **Grep** : Commande de recherche de texte dans des fichiers (`grep "error" log.txt`).

## H
- **Hash** : Empreinte numérique unique générée à partir d'une donnée (ex: mot de passe). Irréversible mathématiquement.
- **Hardening** : Processus de sécurisation d'un système en réduisant sa surface d'attaque.
- **Headless** : Serveur sans interface graphique, géré uniquement en ligne de commande.

## K
- **KVM (Kernel-based Virtual Machine)** : Solution de virtualisation native du noyau Linux.
- **Kernel** : Le cœur du système d'exploitation qui gère les ressources matérielles.

## P
- **Permission (rwx)** : Droits d'accès : Read (Lecture), Write (Écriture), Execute (Exécution).
- **Privilege Escalation** : Action d'élever ses droits d'accès (ex: passer de utilisateur standard à root).

## R
- **Root** : Super-utilisateur sous Linux ayant tous les pouvoirs sur le système.
- **Red Team** : Équipe simulant des attaques pour tester les défenses.
- **Blue Team** : Équipe chargée de la défense et de la détection des attaques.

## S
- **SSH (Secure Shell)** : Protocole crypté pour se connecter à distance de manière sécurisée.
- **Sudo (SuperUser Do)** : Commande permettant d'exécuter une tâche avec les privilèges administrateur.
- **Snapshot** : Image instantanée de l'état d'une machine virtuelle à un instant T (permet de revenir en arrière).

## V
- **Vulnerability (Vulnérabilité)** : Faiblesse dans un système pouvant être exploitée par une menace.
- **VM (Virtual Machine)** : Ordinateur virtuel émulé par un logiciel (hyperviseur).

## W
- **Whoami** : Commande affichant le nom de l'utilisateur courant.
